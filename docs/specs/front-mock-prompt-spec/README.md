# Front Mock Prompt Specification

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

이 문서는 프론트엔드 목업을 생성하기 위한 프롬프트 조립 규격을 정의한다.

이 시스템은 하나의 거대한 프롬프트를 사용하지 않는다.

대신, 여러 개의 md 문서를 조합하여  
완결된 프론트엔드 목업을 생성한다.

---

────────────────────────────────────────────────────────
[핵심 원칙]
────────────────────────────────────────────────────────

1. 프론트는 "문서 세트"로 정의된다
2. 라우터 단위로 분해한다
3. 모든 데이터는 JSON 기반이다
4. 백엔드는 존재하지 않는다
5. 상태 전이는 UI 내부에서 처리한다

---

────────────────────────────────────────────────────────
[프롬프트 구성 요소]
────────────────────────────────────────────────────────

프론트엔드 목업은 아래 문서들을 조합하여 생성한다:

1. PROJECT_META
2. PRD_UI_SUMMARY
3. GLOBAL_RULES
4. ROUTER_MAP
5. STORYBOARD
6. ROUTES/*
7. JSON_CONVENTION
8. GLOBAL_QA

---

────────────────────────────────────────────────────────
[문서 역할]
────────────────────────────────────────────────────────

PROJECT_META  
→ 프로젝트 기본 정보 정의

PRD_UI_SUMMARY  
→ UI 기준 서비스 정의

GLOBAL_RULES  
→ 전체 UI/UX 규칙

ROUTER_MAP  
→ 전체 페이지 구조

STORYBOARD  
→ 사용자 흐름 + 상태 전이

ROUTES  
→ 각 페이지 상세 정의

JSON_CONVENTION  
→ 데이터 구조 규칙

GLOBAL_QA  
→ 전체 검증 기준

---

────────────────────────────────────────────────────────
[프롬프트 조립 방식]
────────────────────────────────────────────────────────

## 전체 생성

아래 순서로 문서를 결합하여 입력한다:

1. PROJECT_META
2. PRD_UI_SUMMARY
3. GLOBAL_RULES
4. ROUTER_MAP
5. STORYBOARD
6. ROUTES (전체)
7. JSON_CONVENTION
8. GLOBAL_QA

---

## 특정 페이지 수정

다음만 사용한다:

- GLOBAL_RULES
- ROUTER_MAP
- 해당 ROUTE md
- 필요 시 STORYBOARD 일부

---

## 라우터 구조 수정

다음만 사용한다:

- GLOBAL_RULES
- ROUTER_MAP

---

────────────────────────────────────────────────────────
[절대 금지]
────────────────────────────────────────────────────────

- 모든 내용을 하나의 프롬프트로 합치기
- 라우터 5개 이상인데 단일 md로 작성
- 백엔드 코드 생성
- API 설계
- DB 설계
- fetch/axios 사용

---

────────────────────────────────────────────────────────
[출력 기대 결과]
────────────────────────────────────────────────────────

AI는 다음을 생성해야 한다:

- React + Vite 프로젝트
- App.tsx 라우팅 구성
- 각 페이지 컴포넌트
- JSON 기반 UI 렌더링
- 상태 전이 처리
- 버튼 이벤트 처리

---

모든 페이지는:

- 실제 서비스처럼 동작해야 한다
- 더미 데이터 기반이어야 한다
- 독립적으로 수정 가능해야 한다

---

────────────────────────────────────────────────────────
[핵심 정의]
────────────────────────────────────────────────────────

이 시스템에서 프론트엔드란:

"UI + 상태 + 흐름이 완결된 제품"

데이터는 이후에 붙는다.

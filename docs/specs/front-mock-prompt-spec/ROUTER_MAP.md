# GLOBAL_RULES

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

이 문서는 프론트엔드 목업 전체에 적용되는 UI/UX 규칙을 정의한다.

모든 페이지는 이 규칙을 반드시 따른다.

이 문서가 없으면 UI가 페이지마다 다르게 생성되어 일관성이 깨진다.

---

────────────────────────────────────────────────────────
[작성 원칙]
────────────────────────────────────────────────────────

- 모든 규칙은 "사용자 경험 기준"으로 정의한다
- 디자인이 아니라 "동작" 중심으로 작성한다
- 모든 페이지에 동일하게 적용 가능해야 한다

---

────────────────────────────────────────────────────────
[1. Layout 규칙]
────────────────────────────────────────────────────────

## 기본 구조

모든 페이지는 아래 구조를 따른다:

- Header
- Main Content
- (선택) Sidebar
- Footer

---

## Header

포함 요소:

- 로고
- 주요 네비게이션
- 사용자 액션 (로그인 / 프로필)

행동:

- 로고 클릭 시 "/" 이동
- 메뉴 클릭 시 해당 라우터 이동

---

## Main Content

- 페이지 핵심 콘텐츠 영역
- 스크롤 가능

---

## Footer

- 최소한의 정보만 포함
- 링크 또는 간단한 안내

---

────────────────────────────────────────────────────────
[2. Navigation 규칙]
────────────────────────────────────────────────────────

- 모든 라우터는 직접 접근 가능해야 한다
- 브라우저 뒤로가기/앞으로가기 정상 동작
- 클릭 시 즉시 라우팅

---

────────────────────────────────────────────────────────
[3. Button 규칙]
────────────────────────────────────────────────────────

## 버튼 유형

- Primary: 주요 액션
- Secondary: 보조 액션
- Ghost: 보조/링크성 액션

---

## 버튼 동작

- 클릭 시 즉시 반응해야 한다
- 로딩 상태 표시 필요

---

## 상태

- default
- hover
- active
- disabled
- loading

---

────────────────────────────────────────────────────────
[4. Input 규칙]
────────────────────────────────────────────────────────

## 기본 동작

- 입력 시 즉시 반영
- validation은 UI에서 처리

---

## 상태

- default
- focus
- error
- disabled

---

## 에러 처리

- 입력값이 잘못되면 즉시 메시지 표시

---

────────────────────────────────────────────────────────
[5. Card / List 규칙]
────────────────────────────────────────────────────────

- 반복 데이터는 카드 또는 리스트로 표현
- 클릭 가능한 요소는 hover 상태 제공

---

────────────────────────────────────────────────────────
[6. 상태 전이 규칙 (핵심)]
────────────────────────────────────────────────────────

모든 화면은 아래 상태를 반드시 가진다:

- initial
- loading
- success
- empty
- error

---

## 상태 정의

initial:
- 진입 직후 상태

loading:
- 데이터 로딩 시
- skeleton 또는 spinner 표시

success:
- 정상 데이터 표시

empty:
- 데이터 없음
- 안내 메시지 표시

error:
- 실패 상태
- 재시도 버튼 제공

---

────────────────────────────────────────────────────────
[7. Interaction 규칙]
────────────────────────────────────────────────────────

- 모든 액션은 즉시 피드백을 준다
- 클릭 → UI 변화 반드시 발생
- 상태 변화 없이 아무 일도 안 일어나면 안 된다

---

────────────────────────────────────────────────────────
[8. Feedback 규칙]
────────────────────────────────────────────────────────

## Toast / Alert

- 주요 액션 후 사용자에게 결과 전달

예:

- 저장 완료
- 삭제 완료
- 에러 발생

---

────────────────────────────────────────────────────────
[9. Loading 규칙]
────────────────────────────────────────────────────────

- 최소 0.3초 이상 표시
- 사용자에게 작업 중임을 명확히 전달

---

────────────────────────────────────────────────────────
[10. Empty 상태 규칙]
────────────────────────────────────────────────────────

- 데이터 없음 이유 설명
- 다음 행동 유도

예:

"데이터가 없습니다 → 추가하기 버튼 제공"

---

────────────────────────────────────────────────────────
[11. Error 상태 규칙]
────────────────────────────────────────────────────────

- 문제 원인 간단히 설명
- 재시도 액션 제공

---

────────────────────────────────────────────────────────

────────────────────────────────────────────────────────

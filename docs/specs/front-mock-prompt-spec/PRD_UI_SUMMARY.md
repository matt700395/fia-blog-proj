# PRD_UI_SUMMARY

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

이 문서는 서비스의 핵심 요구사항을 "UI 관점"에서 요약한다.

이 문서는 전체 PRD를 대체하지 않는다.

대신, 프론트엔드 목업을 생성하기 위해  
UI 설계에 필요한 정보만 압축하여 제공한다.

---

────────────────────────────────────────────────────────
[작성 규칙]
────────────────────────────────────────────────────────

- 기술 구현 내용 금지 (DB, API, 서버)
- UI/UX 기준으로만 작성
- 실제 화면에서 드러나는 요소만 포함
- 모호한 표현 금지

---

────────────────────────────────────────────────────────
[1. What — 무엇을 하는 서비스인가]
────────────────────────────────────────────────────────

서비스를 한 문장으로 정의한다.

형식:

"이 서비스는 ___을/를 할 수 있는 서비스이다."

---

────────────────────────────────────────────────────────
[2. Value — 사용자에게 주는 가치]
────────────────────────────────────────────────────────

사용자가 이 서비스를 통해 얻는 핵심 가치를 정의한다.

형식:

- 사용자는 ___를 빠르게 할 수 있다
- 사용자는 ___를 더 쉽게 관리할 수 있다

---

────────────────────────────────────────────────────────
[3. JTBD — 사용자가 해결하려는 일]
────────────────────────────────────────────────────────

사용자가 이 서비스를 사용하는 이유를 정의한다.

형식:

"사용자는 ___ 상황에서 ___을/를 하기 위해 이 서비스를 사용한다."

---

────────────────────────────────────────────────────────
[4. Primary Personas]
────────────────────────────────────────────────────────

핵심 사용자 2~3명을 정의한다.

형식:

Persona 1:
- 직업 / 상태
- 현재 문제
- 행동 패턴

Persona 2:
- 직업 / 상태
- 현재 문제
- 행동 패턴

---

────────────────────────────────────────────────────────
[5. Core UI Features]
────────────────────────────────────────────────────────

UI 기준 핵심 기능을 정의한다.

형식:

- 기능 1: 사용자가 ___ 할 수 있다
- 기능 2: 사용자가 ___ 할 수 있다

※ 반드시 "화면에서 보이는 기능"만 작성

---

────────────────────────────────────────────────────────
[6. User Interaction Model]
────────────────────────────────────────────────────────

사용자의 주요 인터랙션 패턴을 정의한다.

포함:

- 클릭
- 입력
- 선택
- 필터링
- 탐색

형식:

- 사용자는 ___ 버튼을 클릭한다
- 사용자는 ___을 입력한다
- 사용자는 ___을 선택한다

---

────────────────────────────────────────────────────────
[7. UI States]
────────────────────────────────────────────────────────

모든 주요 상태를 정의한다.

반드시 포함:

- initial
- loading
- success
- empty
- error

형식:

각 상태에서 어떤 UI가 보여야 하는지 설명

---

────────────────────────────────────────────────────────
[8. Non-Goals]
────────────────────────────────────────────────────────

이번 프론트 목업에서 제외할 것

예:

- 실시간 데이터 처리
- 결제 기능
- 관리자 기능

---

────────────────────────────────────────────────────────
[9. MVP Metrics]
────────────────────────────────────────────────────────

프론트 목업 기준 성공 판단 기준

예:

- 모든 주요 흐름이 끊김 없이 이어진다
- 사용자가 주요 기능을 3단계 이내로 수행 가능
- 상태 전이가 자연스럽다

---

────────────────────────────────────────────────────────
[중요 선언]
────────────────────────────────────────────────────────

이 문서는 UI 설계를 위한 문서이다.

- DB 구조를 포함하지 않는다
- API를 정의하지 않는다
- 서버 동작을 가정하지 않는다

모든 설계는 "화면 중심"으로 이루어진다.

---

────────────────────────────────────────────────────────
[AI 수행 규칙]
────────────────────────────────────────────────────────

이 문서를 기반으로:

- 라우터를 설계한다
- 사용자 흐름을 만든다
- 페이지 구성을 정의한다

이 문서에 없는 기능은 생성하지 않는다.

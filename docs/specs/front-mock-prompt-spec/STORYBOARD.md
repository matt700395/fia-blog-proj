# STORYBOARD

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

이 문서는 사용자의 행동 흐름과 상태 전이를 정의한다.

단순한 라우터 나열이 아니라,

"사용자가 어떤 행동을 하고 → 어떤 UI 변화가 발생하며 → 어떤 상태로 전이되는지"

를 명확하게 정의하는 것이 목적이다.

---

────────────────────────────────────────────────────────
[작성 규칙]
────────────────────────────────────────────────────────

- 반드시 "사용자 행동 기준"으로 작성한다
- 이벤트 → UI 변화 → 상태 전이를 포함한다
- 라우터 이동을 명확하게 표시한다
- 단순 페이지 나열 금지

---

────────────────────────────────────────────────────────
[표현 형식]
────────────────────────────────────────────────────────

[현재 라우터]
→ EVENT (사용자 행동)
→ UI 변화
→ 상태 전이
→ [다음 라우터]

---

예시:

"/"
→ CLICK_START_BUTTON
→ 대시보드 로딩 UI 표시
→ loading
→ "/dashboard"

---

────────────────────────────────────────────────────────
[Flow 1 — 핵심 사용자 흐름]
────────────────────────────────────────────────────────

"/"
→ CLICK_MAIN_ACTION
→ CTA 버튼 강조 + 로딩 표시
→ loading
→ "/dashboard"

"/dashboard"
→ VIEW_DATA
→ 카드 리스트 렌더링
→ success

---

────────────────────────────────────────────────────────
[Flow 2 — 검색 / 탐색 흐름]
────────────────────────────────────────────────────────

"/dashboard"
→ INPUT_SEARCH
→ 검색 결과 필터링
→ success

→ CLICK_ITEM
→ 상세 페이지 이동
→ "/item/:id"

"/item/:id"
→ LOAD_DETAIL
→ 상세 정보 표시
→ success

---

────────────────────────────────────────────────────────
[Flow 3 — 인증 흐름]
────────────────────────────────────────────────────────

"/auth"
→ INPUT_EMAIL_PASSWORD
→ 로그인 버튼 활성화
→ initial

→ CLICK_LOGIN
→ 로딩 UI 표시
→ loading

→ LOGIN_SUCCESS
→ "/dashboard"

→ LOGIN_FAIL
→ 에러 메시지 표시
→ error

---

────────────────────────────────────────────────────────
[Flow 4 — Empty 상태 흐름]
────────────────────────────────────────────────────────

"/dashboard"
→ NO_DATA
→ "데이터 없음" 메시지 표시
→ empty

→ CLICK_CREATE
→ 생성 페이지 이동

---

────────────────────────────────────────────────────────
[Flow 5 — Error 처리 흐름]
────────────────────────────────────────────────────────

"/dashboard"
→ API_FAIL (mock)
→ 에러 메시지 표시
→ error

→ CLICK_RETRY
→ 재시도
→ loading

---

────────────────────────────────────────────────────────
[상태 정의]
────────────────────────────────────────────────────────

모든 흐름은 반드시 아래 상태를 포함해야 한다:

- initial
- loading
- success
- empty
- error

---

────────────────────────────────────────────────────────
[체크리스트]
────────────────────────────────────────────────────────

□ 모든 주요 기능이 흐름으로 정의되어 있는가  
□ 상태 전이가 명확하게 정의되어 있는가  
□ 라우터 이동이 누락되지 않았는가  
□ 사용자 행동(EVENT)이 명확한가  

---

────────────────────────────────────────────────────────
[중요 규칙]
────────────────────────────────────────────────────────

이 문서는:

- ROUTES md보다 상위 개념이다
- UI 흐름을 정의하는 기준이다

ROUTES는 이 문서를 따라야 한다.

---

────────────────────────────────────────────────────────
[AI 수행 규칙]
────────────────────────────────────────────────────────

- 모든 페이지는 이 흐름을 기반으로 생성한다
- 정의되지 않은 흐름은 만들지 않는다
- 상태 전이를 반드시 구현한다

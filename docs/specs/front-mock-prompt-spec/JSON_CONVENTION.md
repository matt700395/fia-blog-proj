# JSON_CONVENTION

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

이 문서는 프론트엔드 목업에서 사용하는 JSON 데이터 구조 규칙을 정의한다.

모든 페이지는 이 규칙을 따라 데이터를 구성한다.

이 문서가 없으면 페이지마다 데이터 구조가 달라져 유지보수가 불가능해진다.

---

────────────────────────────────────────────────────────
[핵심 원칙]
────────────────────────────────────────────────────────

1. 라우터 1개 = JSON 1개  
2. 데이터는 해당 라우터 JSON 안에서 완결된다  
3. 다른 라우터와 구조를 맞출 필요 없다  
4. 중복 허용  
5. 백엔드 구조 고려 금지  

---

────────────────────────────────────────────────────────
[파일 위치]
────────────────────────────────────────────────────────

모든 JSON은 아래 경로에 위치한다:

public/data/routes/{route_key}.json

---

예:

- public/data/routes/home.json
- public/data/routes/dashboard.json
- public/data/routes/item_detail.json

---

────────────────────────────────────────────────────────
[기본 구조]
────────────────────────────────────────────────────────

모든 JSON은 아래 구조를 따른다:

{
  "__mock": {
    "mode": "success"
  },
  "page": {},
  "view": {},
  "actions": {}
}

---

────────────────────────────────────────────────────────
[__mock 필드]
────────────────────────────────────────────────────────

목적:
상태 시뮬레이션

값:

- success
- empty
- error
- loading

---

예:

{
  "__mock": {
    "mode": "empty"
  }
}

---

────────────────────────────────────────────────────────
[page]
────────────────────────────────────────────────────────

페이지 메타 정보

예:

{
  "title": "Dashboard",
  "description": "사용자 대시보드"
}

---

────────────────────────────────────────────────────────
[view]
────────────────────────────────────────────────────────

UI 렌더링 데이터

예:

{
  "items": [
    {
      "id": 1,
      "title": "Item 1",
      "description": "설명"
    }
  ]
}

---

────────────────────────────────────────────────────────
[actions]
────────────────────────────────────────────────────────

사용자 행동 정의

예:

{
  "primaryAction": {
    "label": "추가하기",
    "event": "CLICK_CREATE"
  }
}

---

────────────────────────────────────────────────────────
[상태별 처리 규칙]
────────────────────────────────────────────────────────

## success

- 정상 데이터 표시

---

## empty

- 데이터 없음 UI 표시

---

## error

- 에러 메시지 표시
- 재시도 버튼 제공

---

## loading

- skeleton 또는 spinner 표시

---

────────────────────────────────────────────────────────
[데이터 설계 규칙]
────────────────────────────────────────────────────────

- UI에 필요한 데이터만 포함
- DB 구조 고려 금지
- 정규화 금지
- 중복 허용

---

예:

잘못된 방식:

- user 테이블 / post 테이블 분리

올바른 방식:

- 필요한 데이터를 한 JSON 안에 모두 포함

---

────────────────────────────────────────────────────────
[JSON Import 규칙]
────────────────────────────────────────────────────────

Vite 기준:

```ts
import data from "../../public/data/routes/dashboard.json";
```

---

또는 fetch 사용 금지

---

────────────────────────────────────────────────────────
[중요 규칙]
────────────────────────────────────────────────────────

* JSON은 "UI를 위한 데이터"이다
* 서버 데이터를 흉내내지 않는다
* API 응답 형태를 따라하지 않는다

---

────────────────────────────────────────────────────────
[AI 수행 규칙]
────────────────────────────────────────────────────────

* 모든 페이지는 JSON 기반으로 렌더링한다
* 상태(__mock.mode)에 따라 UI를 분기한다
* 데이터 부족 시 JSON에 추가한다 (코드에서 생성 금지)

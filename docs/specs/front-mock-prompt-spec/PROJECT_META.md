# PROJECT_META

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

이 문서는 프론트엔드 목업의 "최상위 컨텍스트"를 정의한다.

모든 UI/UX 설계, 라우터 구조, 사용자 흐름은  
이 문서를 기준으로 해석된다.

이 문서가 모호하면 전체 프론트 설계가 무너진다.

---

────────────────────────────────────────────────────────
[작성 규칙]
────────────────────────────────────────────────────────

- 기술적인 내용 작성 금지 (DB, API, 서버 등)
- UI 관점에서만 작성
- 구체적으로 작성 (모호한 표현 금지)
- 실제 사용자 기준으로 작성

---

────────────────────────────────────────────────────────
[1. 프로젝트명]
────────────────────────────────────────────────────────

- 서비스 이름
- 내부 코드명 (선택)

---

────────────────────────────────────────────────────────
[2. 목적 (Purpose)]
────────────────────────────────────────────────────────

이 서비스가 해결하려는 문제를 한 문장으로 정의한다.

형식:

"이 서비스는 ___ 문제를 해결하기 위한 ___이다."

---

────────────────────────────────────────────────────────
[3. 서비스 유형]
────────────────────────────────────────────────────────

다음 중 선택 또는 조합:

- SaaS
- Marketplace
- Community
- Tool
- Dashboard
- Content Platform

---

────────────────────────────────────────────────────────
[4. MVP 범위]
────────────────────────────────────────────────────────

이번 프론트 목업에서 구현할 범위를 정의한다.

포함:

- 반드시 포함되는 기능
- 제외되는 기능

형식:

포함:
- 기능 A
- 기능 B

제외:
- 기능 C
- 기능 D

---

────────────────────────────────────────────────────────
[5. 핵심 사용자]
────────────────────────────────────────────────────────

- 주요 사용자 1~2명 정의
- 구체적인 상황 포함

형식:

사용자 1:
- 누구인지
- 어떤 상황에서 사용하는지
- 어떤 목표를 가지고 있는지

---

────────────────────────────────────────────────────────
[6. 핵심 시나리오]
────────────────────────────────────────────────────────

사용자가 이 서비스를 사용하는 대표 흐름을 서술한다.

형식:

"사용자는 ___ 하기 위해 ___ 페이지에 들어와서  
___ 행동을 하고  
결과적으로 ___를 얻는다."

---

────────────────────────────────────────────────────────
[7. 비목표 (Non-Goals)]
────────────────────────────────────────────────────────

이번 단계에서 절대 하지 않는 것 명시

예:

- 실시간 데이터 연동
- 결제 기능
- 관리자 기능

---

────────────────────────────────────────────────────────
[8. 성공 기준]
────────────────────────────────────────────────────────

프론트 목업이 완성되었다고 판단하는 기준

예:

- 모든 라우터 진입 가능
- 상태 전이 정상 동작
- 사용자 흐름 완성
- UI 끊김 없음

---

────────────────────────────────────────────────────────
[중요 선언]
────────────────────────────────────────────────────────

이 프로젝트는 프론트엔드 목업 단계이다.

- 서버를 생성하지 않는다
- DB를 설계하지 않는다
- API를 만들지 않는다
- 외부 요청을 하지 않는다

모든 데이터는 로컬 JSON 기반이다.
모든 인터랙션은 UI 상태 변화로 처리한다.

---

────────────────────────────────────────────────────────
[AI 수행 규칙]
────────────────────────────────────────────────────────

이 문서를 기반으로:

- 라우터를 정의한다
- 사용자 흐름을 설계한다
- UI 구조를 결정한다

이 문서와 충돌하는 설계는 금지한다.

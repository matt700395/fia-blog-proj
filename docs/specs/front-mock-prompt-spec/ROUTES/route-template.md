# ROUTE_TEMPLATE

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

이 문서는 개별 라우터 페이지를 정의하기 위한 템플릿이다.

모든 ROUTES/*.md는 이 구조를 따라 작성된다.

---

────────────────────────────────────────────────────────
[작성 규칙]
────────────────────────────────────────────────────────

- 하나의 md는 하나의 라우터만 정의한다
- ROUTER_MAP의 route_key와 반드시 일치해야 한다
- JSON_CONVENTION을 따른다
- GLOBAL_RULES를 따른다
- STORYBOARD 흐름을 반영한다

---

────────────────────────────────────────────────────────
[기본 정보]
────────────────────────────────────────────────────────

Route Path:
예: /dashboard

Route Key:
예: dashboard

Access:
public / protected

---

────────────────────────────────────────────────────────
[1. 목적]
────────────────────────────────────────────────────────

이 페이지가 존재하는 이유를 정의한다.

예:

"사용자가 자신의 데이터를 한눈에 확인하고 주요 액션을 수행할 수 있는 페이지"

---

────────────────────────────────────────────────────────
[2. 진입 조건]
────────────────────────────────────────────────────────

이 페이지에 들어오는 조건 정의

예:

- 로그인 상태여야 한다
- 특정 버튼 클릭 후 진입

---

────────────────────────────────────────────────────────
[3. 레이아웃 구조]
────────────────────────────────────────────────────────

위 → 아래 순서로 작성

예:

1. Header
2. 필터 영역
3. 카드 리스트
4. CTA 버튼

---

────────────────────────────────────────────────────────
[4. JSON 참조]
────────────────────────────────────────────────────────

파일 위치:

public/data/routes/{route_key}.json

---

사용 데이터:

- view.items
- view.filters
- actions.primaryAction

---

────────────────────────────────────────────────────────
[5. UI 구성 요소]
────────────────────────────────────────────────────────

각 UI 요소 정의

예:

- 검색 input
- 필터 dropdown
- 카드 리스트
- 버튼

---

────────────────────────────────────────────────────────
[6. 이벤트 정의]
────────────────────────────────────────────────────────

EVENT 이름과 동작 정의

예:

CLICK_ITEM  
→ 상세 페이지 이동

CLICK_CREATE  
→ 생성 페이지 이동

INPUT_SEARCH  
→ 리스트 필터링

---

────────────────────────────────────────────────────────
[7. 상태 머신]
────────────────────────────────────────────────────────

반드시 포함:

- initial
- loading
- success
- empty
- error

---

각 상태 설명:

initial:
- 페이지 진입 상태

loading:
- skeleton 또는 spinner 표시

success:
- 데이터 정상 표시

empty:
- 데이터 없음 메시지 표시

error:
- 에러 메시지 + 재시도 버튼

---

────────────────────────────────────────────────────────
[8. 사용자 행동 흐름]
────────────────────────────────────────────────────────

STORYBOARD 기반으로 작성

예:

"/dashboard"
→ CLICK_ITEM
→ "/item/:id"

────────────────────────────────────────────────────────
[체크리스트]
────────────────────────────────────────────────────────

□ ROUTER_MAP과 일치하는가  
□ JSON 참조가 명확한가  
□ 상태 전이가 정의되었는가  
□ 이벤트가 누락되지 않았는가  
□ GLOBAL_RULES를 따르는가

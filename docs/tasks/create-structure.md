# Task: CREATE_STRUCTURE

Task Code: T5  
Phase: P2 — STRUCTURE  

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

프론트 설계(front-prompt)와 PRD를 기반으로  
데이터 구조(IA, Entity, ERD, Table Schema)를 정의한다.

이 작업은:

"UI → 데이터 구조 추출"

단계이다.

---

────────────────────────────────────────────────────────
[입력]
────────────────────────────────────────────────────────

필수 입력:

- docs/system.md
- docs/task-definition.md
- docs/current/front-prompt/
- docs/current/prd.md
- docs/tasks/create-structure.md
- docs/specs/supabase-spec.md

선택 입력:

- docs/current/change-log.md

---

────────────────────────────────────────────────────────
[출력]
────────────────────────────────────────────────────────

- docs/current/structure.md

포함 내용:

1. IA (Information Architecture)
2. Entity 정의
3. ERD (관계)
4. Table Schema (컬럼)

- docs/current/schema.sql

포함 내용 (Supabase SQL Editor 실행용):

1. CREATE TABLE (의존성 순서)
2. FK 제약 조건
3. RLS 활성화 및 Policy
4. Index (필요 시)

---

────────────────────────────────────────────────────────
[작업 절차]
────────────────────────────────────────────────────────

1. 라우터 기반 IA 추출
   - ROUTER_MAP 분석
   - ROUTES/*.md 분석
   - 페이지 단위 기능 그룹핑

2. 데이터 객체 식별
   - 화면에 등장하는 데이터 요소 추출
   - 반복되는 데이터 패턴 식별

3. Entity 정의
   - 핵심 객체 정의 (예: User, Item, Order)
   - 속성 정의

4. 관계 정의
   - 1:N, N:N 관계 정의
   - 참조 관계 명확화

5. ERD 설계
   - Entity 간 관계 시각화

6. Table Schema 작성
   - 테이블명
   - 컬럼명
   - 타입
   - PK / FK 정의

7. schema.sql 작성 (supabase-spec.md 기준)
   - CREATE TABLE 문 작성
   - FK 제약 조건 추가
   - RLS 활성화 및 Policy 작성
   - 필요한 Index 추가
   - Supabase SQL Editor에서 바로 실행 가능한 형태로 작성

---

────────────────────────────────────────────────────────
[중요 규칙]
────────────────────────────────────────────────────────

- 반드시 "프론트 기준"으로 설계한다
- UI에 없는 데이터는 만들지 않는다
- JSON 구조를 그대로 DB로 옮기지 않는다
- 필요한 경우 정규화 수행한다

---

────────────────────────────────────────────────────────
[설계 기준]
────────────────────────────────────────────────────────

- 중복 최소화
- 명확한 관계
- 확장 가능

나쁜 구조:

- JSON 그대로 DB화
- 의미 없는 테이블 분리
- 관계 없는 테이블

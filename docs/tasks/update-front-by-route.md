# Task: UPDATE_FRONT_BY_ROUTE

Task Code: T2  
Phase: P1 — FRONT_MOCK  

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

특정 라우터(페이지)의 UI/UX를 수정하거나 구현한다.

이 작업은 전체 프론트를 다시 생성하지 않고  
"부분 수정"을 수행하는 것이 핵심이다.

---

────────────────────────────────────────────────────────
[입력]
────────────────────────────────────────────────────────

필수 문서:

- docs/system.md
- docs/task-definition.md
- docs/current/front-prompt/
- docs/tasks/update-front-by-route.md

선택 입력:

- docs/current/change-log.md
- 특정 ROUTES/*.md

---

────────────────────────────────────────────────────────
[출력]
────────────────────────────────────────────────────────

- 수정된 프론트 코드
- 필요 시 docs/current/front-prompt/ 일부 업데이트

---

────────────────────────────────────────────────────────
[작업 절차]
────────────────────────────────────────────────────────

1. 수정 대상 라우터 식별
   - route_key 확인
   - path 확인

2. 관련 문서 로드
   - ROUTES/{route_key}.md
   - STORYBOARD.md (해당 흐름 부분)
   - GLOBAL_RULES.md

3. 수정 유형 판단

   - UI 변경 (레이아웃, 디자인)
   - UX 변경 (흐름, 인터랙션)
   - 상태 변경 (loading, empty 등)

4. 프론트 코드 수정
   - React 컴포넌트 수정
   - 상태 처리 수정
   - 이벤트 처리 수정

5. 필요 시 문서 반영
   - ROUTES md 수정
   - STORYBOARD 일부 수정

---

────────────────────────────────────────────────────────
[수정 범위 규칙]
────────────────────────────────────────────────────────

허용:

- 특정 페이지 UI 변경
- 사용자 흐름 개선
- 상태 처리 수정

금지:

- 전체 front 재생성
- ROUTER_MAP 변경 (특수한 경우 제외)
- GLOBAL_RULES 임의 변경
- 다른 라우터 영향 수정

────────────────────────────────────────────────────────

- 변경은 최소 범위로 제한한다
- 다른 페이지에 영향이 없어야 한다

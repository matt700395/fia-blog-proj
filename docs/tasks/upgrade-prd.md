# Task: UPGRADE_PRD

Task Code: T4  
Phase: P1 — FRONT_MOCK  

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

업데이트된 front-prompt 문서 세트를 기준으로  
현재 PRD를 최신 상태로 재정의한다.

이 작업은:

"프론트 → PRD 역설계"

단계이다.

---

────────────────────────────────────────────────────────
[입력]
────────────────────────────────────────────────────────

필수 입력:

- docs/current/front-prompt/
- 기존 docs/current/prd.md

선택 입력:

- docs/current/change-log.md

---

────────────────────────────────────────────────────────
[출력]
────────────────────────────────────────────────────────

- docs/current/prd.md (업데이트)

---

────────────────────────────────────────────────────────
[작업 절차]
────────────────────────────────────────────────────────

1. front-prompt 분석

   - PROJECT_META 확인
   - PRD_UI_SUMMARY 확인
   - ROUTER_MAP 확인
   - STORYBOARD 확인

2. 변경된 기능 식별

   - 추가된 기능
   - 수정된 기능
   - 삭제된 기능

3. 사용자 흐름 재정의

   - STORYBOARD 기준으로 흐름 재작성

4. PRD 업데이트

   - 기능 정의 업데이트
   - 사용자 흐름 업데이트
   - MVP 범위 수정

5. 변경 이력 반영

   - change-log 기반 정리

---

────────────────────────────────────────────────────────
[중요 규칙]
────────────────────────────────────────────────────────

- 반드시 "프론트 기준"으로 PRD를 작성한다
- PRD가 프론트를 설명하는 문서여야 한다
- 프론트에 없는 기능을 PRD에 추가하지 않는다

- 프론트에 없는 기능 추가

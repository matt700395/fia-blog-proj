# Task: TRANSITION_1_TO_2

Task Code: T7  
Phase: P2 — STRUCTURE  

────────────────────────────────────────────────────────
[목적]
────────────────────────────────────────────────────────

프론트 목업 단계(P1)가 충분히 안정화되었는지 검증하고  
구조 설계 단계(P2)로 전환한다.

이 작업은:

"프론트 완성 → 데이터 구조 설계 시작"

을 결정하는 게이트 역할이다.

- docs/tasks/transition/1-to-2.md

────────────────────────────────────────────────────────
[출력]
────────────────────────────────────────────────────────

- P2 진입 여부 판단
- docs/current/change-log.md 업데이트 (선택)

---

────────────────────────────────────────────────────────
[작업 절차]
────────────────────────────────────────────────────────

1. 프론트 완성도 검증

   GLOBAL_QA 기준으로 확인:

   - 모든 라우터 정상 동작
   - 상태 전이 존재
   - 주요 사용자 흐름 완성
   - UI 피드백 정상

2. 구조 설계 가능 여부 판단

   다음 질문에 답한다:

   - 모든 주요 화면이 확정되었는가?
   - 라우터 변경 가능성이 낮은가?
   - UX 흐름이 안정적인가?

3. 전환 결정

   - YES → P2 진행
   - NO → P1 유지 (T2/T3 반복)

---

────────────────────────────────────────────────────────
[전환 기준 — 반드시 충족]
────────────────────────────────────────────────────────

□ 라우터 구조 확정됨  
□ 주요 UX 흐름 확정됨  
□ 상태 전이 모두 정의됨  
□ UI 변경 가능성 낮음  

---

────────────────────────────────────────────────────────
[전환 금지 조건]
────────────────────────────────────────────────────────

다음 중 하나라도 해당하면 전환 금지:

- UI 계속 변경 중
- 라우터 계속 수정 중
- UX 흐름 불안정
- QA 실패 존재

---

────────────────────────────────────────────────────────
[중요 규칙]
────────────────────────────────────────────────────────

- 프론트가 "완결 수준"이 아닐 경우 절대 P2로 넘어가지 않는다
- DB 구조는 프론트를 기준으로 설계해야 한다
- 성급한 전환은 이후 모든 작업을 무너뜨린다

---

────────────────────────────────────────────────────────
[브랜치 전환 절차]
────────────────────────────────────────────────────────

전환 판단 YES 시:

1. 현재 p1 상태 태그
   git tag v{N}-p1-done

2. p2 브랜치로 전환 및 p1 내용 반영
   git checkout p2
   git merge p1

3. docs/current/VERSION.md 업데이트
   Phase: P2

전환 판단 NO 시:

- 브랜치 변경 없음
- p1 에서 T2 / T3 계속 수행

────────────────────────────────────────────────────────
[작업 절차]
────────────────────────────────────────────────────────

1. 구조 완성도 검증

   structure.md 기준으로 확인:

   - 모든 주요 Entity 정의됨
   - ERD 관계 명확함
   - Table Schema 존재
   - 데이터 흐름 설명 가능

2. 프론트 대응 가능 여부 검증

   다음 질문에 답한다:

   - 모든 화면에서 필요한 데이터 정의되었는가?
   - 누락된 데이터가 없는가?
   - 불필요한 Entity가 없는가?

3. API 설계 가능 여부 판단

   - CRUD 기준으로 API 정의 가능한가?
   - 데이터 흐름이 자연스러운가?

4. 전환 결정

   - YES → P3 진행
   - NO → P2 유지 (T5 반복)

---

────────────────────────────────────────────────────────
[전환 기준 — 반드시 충족]
────────────────────────────────────────────────────────

□ 모든 주요 Entity 정의됨  
□ ERD 관계 명확함  
□ 테이블 구조 완성됨  
□ 프론트 요구사항 충족

────────────────────────────────────────────────────────
[전환 금지 조건]
────────────────────────────────────────────────────────

다음 중 하나라도 해당하면 전환 금지:

- Entity 정의 부족
- 관계 불명확
- 테이블 구조 불완전
- 데이터 흐름 설명 불가

---

────────────────────────────────────────────────────────
[중요 규칙]
────────────────────────────────────────────────────────

- 구조가 완성되지 않으면 API 설계 금지
- 구조 변경 가능성이 있으면 전환 금지
- API는 구조를 그대로 반영해야 한다

────────────────────────────────────────────────────────

   git tag v{N}-p2-done

   Phase: P3

- p2 에서 T5 계속 수행
